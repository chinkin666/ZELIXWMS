技术式样书

1. 系统架构

Frontend (Vue 3 / Vite)
  -> /api/*
Backend (Express / Node.js)
  -> Mongoose
MongoDB

外部连携:
  -> Yamato B2 API / Proxy
  -> 本地文件上传目录 / 渲染输出
层级技术职责
前端Vue 3、TypeScript、Pinia、Vue Router、Konva页面交互、数据编辑、模板设计、打印预览、业务作业。
后端Node.js 20、Express 4、TypeScript、Mongoose、PinoREST API、订单处理、模板服务、渲染、外部接口联动。
数据库MongoDB 7订单、商品、设置、模板、自动规则与日志持久化。
共享层shared workspace类型定义、转换类型、打印单位等共用能力。

2. 目录结构

目录说明
frontend/src/views业务页面，包括出荷指示、作业、结果、设置。
frontend/src/components表格、导入、打印、自动处理、运单相关组件。
backend/src/api路由与控制器。
backend/src/modelsMongoDB 模型定义。
backend/src/services业务服务，包括渲染、Yamato B2、自动处理等。
backend/src/scriptsseed、dump、restore、migration 脚本。

3. 接口式样

模块基础路径主要能力
Shipment Orders/api/shipment-orders列表、详情、手工批量创建、状态变更、回执导入、批量更新/删除。
Products/api/products商品维护、SKU 解析、导入校验、批量导入。
Mapping Configs/api/mapping-configs导入/转换映射定义管理。
Carriers/api/carriers配送业者及格式定义维护。
Print Templates/api/print-templates标签模板维护与渲染依赖。
Form Templates/api/form-templates帐票模板维护。
Carrier Automation/api/carrier-automationYamato B2 配置、验证、导出、打印、取回 PDF、取消确认。
Order Groups/api/order-groups检品分组管理。
Auto Processing Rules/api/auto-processing-rules自动处理规则管理、排序、手工运行。
Render/api/render打印模板渲染服务。

4. 核心数据模型

4.1 订单状态

{
  status: {
    carrierReceipt: { isReceived, receivedAt },
    confirm: { isConfirmed, confirmedAt },
    printed: { isPrinted, printedAt },
    shipped: { isShipped, shippedAt }
  }
}

4.2 主要集合

  • orders: 出荷订单主表，含收件人、发件人、商品、状态、原始行、配送附加数据。
  • products: 商品主数据，支持主 SKU/子 SKU、条码、温层、价格、图片。
  • mapping_configs: Excel/CSV 到内部结构的映射和转换流水线。
  • print_templates / form_templates: 标签与帐票模板。
  • carrier_automation_configs: 配送自动化参数，主要用于 Yamato B2。
  • auto_processing_rules / auto_processing_logs: 自动规则与执行日志。
