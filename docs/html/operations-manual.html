操作说明书

面向仓库现场、出荷担当、主数据维护人员。操作路径依据 frontend/src/router/index.ts 与页面组件整理。

功能说明书

1. 菜单结构

一级菜单二级功能用途
首页/home展示系统入口与快捷跳转。
商品管理/products/list维护商品、子 SKU、条码、价格、温层和图片。
出荷指示/shipment-orders/create、/shipment-orders/confirm创建出荷数据、检查并确认待出荷订单。
出荷作业/shipment-operations/tasks、/shipment-operations/list、1-1/N-1 检品基于已确认且已取得运单信息的订单执行检品与出荷。

2. 日常作业步骤

2.1 出荷指示作成

  1. 进入 出荷指示作成 页面。
  2. 导入订单文件或手工录入订单。
  3. 系统按映射配置解析原始列并生成订单行。
  4. 补充依頼主、配送业者、商品信息、配送方式等必要字段。
  5. 如使用 Yamato B2，可先执行验证，再执行导出取得 trackingId。

2.2 出荷指示确认

  1. 进入 /shipment-orders/confirm。
  2. 筛选未确认订单。
  3. 检查收件人、商品、配送方式、依頼主、时间指定等关键项。
  4. 执行单笔或批量确认，系统将写入 status.confirm.isConfirmed=true。

2.3 出荷作业与检品

  1. 进入作业一览，系统默认关注“已确认、已取得运单、未出荷”的订单。
  2. 根据作业方式选择 1-1 检品或 N-1 检品。
  3. 扫描商品条码，核对 SKU、数量、打印状态。
  4. 必要时打印运单或帐票。
  5. 完成后更新出荷状态。

3. 设置维护说明

设置项典型内容影响范围
依頼主设置发货人名称、地址、电话、ヤマト发店代码订单默认 sender、Yamato B2 出力。
配送业者设置业者代码、名称、导入格式、tracking 列回执导入、运单生成、打印模板绑定。
映射模式外部 Excel/CSV 列到内部订单字段的映射规则订单导入质量与自动解析结果。
打印模板标签尺寸、元素布局、字段绑定运单、标签、贴纸打印。
帐票模板列布局、页边距、页眉页脚、条码拣货单、明细单、作业单输出。
自动处理条件、动作、优先级、手动/自动触发订单分组、自动加商品等后处理。

4. 常见操作要点

  • 订单状态常用标记为：确认、运单回执取得、已打印、已出荷。
  • Yamato B2 相关功能先校验，再导出，再批量取 PDF/打印，异常时可执行 unconfirm 回退。
  • 商品主数据维护时，应优先保证主 SKU、子 SKU、条码唯一和可解析。
  • 映射配置变更后，应使用样例文件重新验证导入结果。
