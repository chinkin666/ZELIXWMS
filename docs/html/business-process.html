业务流程说明书

1. 总体流程

外部订单文件/手工录入
  -> 映射转换
  -> 生成出荷订单
  -> 订单确认
  -> 运单验证/导出/回执取得
  -> 检品与打印
  -> 出荷完成
  -> 出荷实绩查询

2. 订单导入流程

  1. 业务人员准备来自 EC 或外部系统的订单文件。
  2. 系统根据映射配置将原始列转换为内部订单字段。
  3. 订单写入 orders 集合，自动生成 orderNumber 和 _productsMeta。
  4. 若配置了自动处理规则，系统按触发事件执行后处理。

3. 出荷确认与运单流程

阶段关键条件主要动作
确认前未确认、可能保留、可能无 trackingId检查地址、商品、依頼主、配送方式。
确认status.confirm.isConfirmed=true锁定该批订单进入后续作业。
B2 验证Yamato B2 已配置调用 validate 接口校验必填和格式。
B2 导出验证通过向外部 API 送信并回写 trackingId。
回执取得导出成功或回执导入status.carrierReceipt.isReceived=true。

4. 检品与出荷流程

  1. 系统筛选“已确认 + 已取得运单 + 未出荷”订单进入作业列表。
  2. 现场根据检品组、订单类型选择 1-1 或 N-1 检品方式。
  3. 扫描商品条码，系统通过商品主档和子 SKU 进行匹配。
  4. 必要时打印标签、帐票或获取 Yamato 批量 PDF。
  5. 完成后写入 status.printed.isPrinted 和 status.shipped.isShipped。

5. 异常回退流程

  • Yamato B2 异常时，可执行 yamatoB2Unconfirm 回退。
  • 取消确认时，系统可将 confirm、printed、carrierReceipt、shipped 等状态复位。
  • 自动处理规则执行失败不会阻断全局流程，但会记录失败日志供追踪。

6. 状态流转定义

新建订单
  -> confirm
  -> carrierReceipt
  -> printed
  -> shipped

支持逆向操作:
  shipped/printed/carrierReceipt/confirm
  <- unconfirm or carrier-specific rollback
