环境搭建手册

1. 运行前提

组件版本要求说明
Node.js>= 20根目录与前后端 package.json 均要求 Node 20+。
npm>= 10用于 workspace 安装和脚本执行。
MongoDB>= 7本地开发默认连接 mongodb://127.0.0.1:27017/nexand-shipment。

2. 安装步骤

cd d:\GitHub\ZELIXWMS
npm install

cd backend
copy env.example .env
cd ..

根项目通过 workspaces 管理依赖，postinstall 会自动对子项目执行 bootstrap。

3. 环境变量

3.1 后端

NODE_ENV=development
PORT=4000
HOST=0.0.0.0
FRONTEND_URL=http://localhost:4001
MONGODB_URI=mongodb://127.0.0.1:27017/nexand-shipment
MONGODB_DB=nexand-shipment
FILE_STORAGE_DIR=./tmp/uploads

3.2 前端

VITE_BACKEND_ORIGIN=http://localhost:4000
VITE_BACKEND_API_PREFIX=/api

4. 本地启动

npm run db
npm run dev:backend
npm run dev:frontend
服务默认端口说明
MongoDB27017数据库实例，数据目录默认 local-db/mongo。
Backend API4000Express 服务，包含 /health 和 /api/*。
Frontend5173 或 Vite 默认开发服务器，页面发起 API 请求到后端。

5. 数据初始化与备份恢复

npm run seed
npm run db:dump
npm run db:restore
  • seed 用于初始化配送业者、模板、映射等基础数据。
  • db:dump 将数据库导出到 local-db/dump。
  • db:restore 从 local-db/dump 恢复数据。

6. 常见排错

现象排查点
前端无法访问 API检查 VITE_BACKEND_ORIGIN、后端端口、CORS 中的 FRONTEND_URL。
后端启动失败检查 MongoDB 是否启动，.env 是否存在，端口是否冲突。
Yamato B2 失败检查 carrier automation 配置、API key、客户代码、代理服务可达性。
打印异常检查模板绑定字段、字体文件、render 服务和 PDF 缓存。
