《ZELIX WMS 库存锁设计（Inventory Lock Architecture）》。
这是 高并发仓库系统最关键的技术点之一，因为：

如果库存锁设计不好
系统一定会出现：
超卖
库存不一致
订单冲突

很多大型仓储系统（例如 SAP Extended Warehouse Management、Manhattan WMS）都会设计专门的 库存锁机制。

库存变化不是直接扣库存，而是：

锁定库存 → 发货 → 扣库存
二、库存状态模型

库存必须分成两部分：

available

示例：

SKU123

available = 100
reserved = 0

订单进入：

订单需要 2

库存锁定：

available = 98
reserved = 2

库存锁状态：

六、库存锁释放

如果订单取消：

release reservation

库存恢复：

available +2
reserved -2

更新：

reservation status = released

十、库存锁并发控制

避免：

两个订单同时扣库存

Atomic Update

db.inventory.updateOne(

  {
