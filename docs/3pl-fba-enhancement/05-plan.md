# 05 开发计划 / 開発計画

> 3PL + FBA 通过型业务系统增强 / 3PL + FBA 通過型業務システム強化

## 1. Phase 总览 / Phase 概要

> **核心原则**: 先跑通 FBA 通过型一条线，再扩展。
> **コア原則**: まず FBA 通過型を一本通す、その後拡張。

```
Phase 0 (基础)    ─── 5-7 天  ─── 客户模型改造 + 照片存储 + 门户骨架
Phase 1 (通过型)  ─── 15-20 天 ─── 客户门户入库预定 + 仓库端受付/作业/出货 + PDF拆分
Phase 2 (增强)    ─── 10-12 天 ─── 检品增强 + 贴标管理 + 异常报告 + 箱级管理
Phase 3 (扩展)    ─── 8-10 天 ─── RSL/B2B 适配 + 保管型 + 循环盘点 + 库龄预警
Phase 4 (管理)    ─── 10-14 天 ─── KPI + 报告 + 计费扩展 + 大促 + 审计
```

## 3. Phase 0 + Phase 1 详细排期 / Phase 0 + Phase 1 詳細スケジュール

### Phase 0: 基础（5-7 天）

| 顺序 | 任务 | 预估 | 依赖 | 产出 |
|------|------|------|------|------|
| 0.1 | 客户模型改造 | 2-3 天 | 无 | SubClient + Shop + PriceCatalog + Client 扩展 |
| 0.2 | 照片存储基础设施 | 1-2 天 | 无 | photoService + MinIO + PhotoCapture 组件 |
| 0.3 | 门户项目骨架 | 2 天 | 无 | portal/ 项目初始化 + 路由 + 登录 + i18n |

**0.1 和 0.2 可并行，0.3 在 0.1 之后（登录需要客户模型）**

### Phase 1: FBA 通过型跑通（15-20 天）

| 顺序 | 任务 | 预估 | 依赖 | 产出 |
|------|------|------|------|------|
| 1.1 | 门户-商品管理 | 2-3 天 | Phase 0 | 商品 CRUD + Excel 导入 + 标签打印 |
| 1.2 | 门户-入库预定创建 | 4-5 天 | 1.1 | 5 步向导 + 作业选项 + 费用预估 + FBA 标上传 |
| 1.3 | 后端-入库预定 API | 3-4 天 | Phase 0 | 通过型 InboundOrder API + 作业选项 + 计费 |
| 1.4 | 后端-FBA 标 PDF 拆分 | 2-3 天 | 0.2 | PDF 上传 + 4/6-up 拆分 + 单张存储 |
| 1.5 | 仓库端-受付+作业+出货 | 3-4 天 | 1.3, 1.4 | 扫码受付 + 差异 + 作业执行 + 出货匹配 |
| 1.6 | 门户-预定详情+费用 | 2-3 天 | 1.3 | 进度查看 + 差异确认 + 费用明细 |

**1.1 和 1.3 可并行，1.2 和 1.4 可并行**

## 4. 里程碑 / マイルストーン

| 里程碑 | 完成标志 | 预计时间 |
|--------|---------|---------|
| M0: 基础就绪 | 客户模型 + 照片存储 + 门户骨架能登录 | Phase 0 结束 |
| M1: 门户可用 | 客户能登录门户、登录商品、创建入库预定 | Phase 1 第 7 天 |
| M2: FBA 一条线跑通 | 客户预定→仓库受付→作业→PDF拆分→出货→追踪号→费用 | Phase 1 结束 |
| M3: 功能增强 | 多维检品 + 贴标管理 + 异常报告 + 箱级管理 | Phase 2 结束 |
| M4: 多场景 | RSL + B2B + 保管型 + 盘点 | Phase 3 结束 |
| M5: 管理闭环 | KPI + 报告 + 计费扩展 | Phase 4 结束 |

## 5. 风险与应对 / リスクと対策

| 风险 | 影响 | 应对 |
|------|------|------|
| 照片存储方案选型延迟 | 阻塞 Phase 1 所有照片相关功能 | 先用本地文件系统 prototype，后切 S3 |
| 热敏打印机集成困难 | 贴标流程无法闭环 | 先支持 PDF 导出手动打印，后接 ESC/POS |
| 移动端适配工作量超预期 | 仓库现场无法使用 | 先做 PC 版，移动端作为 Phase 1.5 补充 |
| Amazon FBA 规格频繁变更 | 箱规校验逻辑需反复调整 | 校验规则配置化，不硬编码 |
