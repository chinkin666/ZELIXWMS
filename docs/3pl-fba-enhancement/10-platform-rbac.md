### 3.3 权限实现方案 / 権限実装方案

```
数据模型:

Role {
  _id: ObjectId
  tenantId: string
  code: string                         // 'warehouse_admin', 'receiver', etc.
  name: string                         // 显示名（多语言）
  permissions: [string]                // 权限标识数组
  isSystem: boolean                    // 系统预设角色不可删除
  warehouseId: ObjectId                // 可选：仓库级角色
}

// 权限标识格式: resource:action
// 権限識別子形式: リソース:アクション
permissions 示例:
  'inbound:view'
  'inbound:receive'
  'inbound:variance'
  'inspection:operate'
  'inspection:verify'
  'labeling:operate'
  'labeling:verify'
  'shipping:operate'
  'shipping:tracking'
  'exception:create'
  'exception:resolve'
  'inventory:view'
  'inventory:adjust'
  'cycle_count:operate'
  'staff:manage'
  'report:view'

User 扩展:
+ roles: [ObjectId]                    // 可多角色
+ warehouseId: ObjectId                // 所属仓库
+ assignedWarehouses: [ObjectId]       // 可操作的仓库列表

后端校验:
// middleware
function requirePermission(permission: string) {
  return (req, res, next) => {
    if (!req.user.permissions.includes(permission)) {
      return res.status(403).json({ error: 'Permission denied' })
    }
    next()
  }
}

// 路由使用
router.post('/inbound/:id/receive',
  requireAuth,
  requirePermission('inbound:receive'),
  controller.receive
)

前端校验:
// composable
const { hasPermission } = useAuth()

// 模板中
<button v-if="hasPermission('inbound:receive')">受付</button>

// 路由守卫
{ path: '/receive', meta: { permission: 'inbound:receive' } }
```

## 4. 客户端权限 / 顧客端権限

```
顶层客户管理员:
  - 全部子客户/店铺的数据
  - 创建/管理子客户和店铺
  - 查看汇总账单 + 全部明细
  - 创建子客户门户账号
  - 所有店铺的入库预定

子客户用户:
  - 只看自己 + 自己店铺的数据
  - 创建自己店铺的入库预定
  - 查看自己的费用明细
  - 不能看其他子客户
  - 不能看汇总账单

实现:
  用户登录时:
    if (user.subClientId) {
      // 子客户用户 → 所有查询自动加 subClientId 过滤
    } else if (user.clientId) {
      // 顶层管理员 → 按 clientId 过滤，能看所有子客户
    }
```
