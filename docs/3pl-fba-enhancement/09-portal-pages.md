# 09 客户门户页面设计 / 顧客ポータル画面設計

> 客户门户是独立前端应用（portal/），与仓库管理端（frontend/）分离。
> 顧客ポータルは独立フロントエンドアプリ（portal/）、倉庫管理端（frontend/）とは分離。

## 1. 应用架构 / アプリケーション構成

```
ZELIX WMS
  ├─ frontend/          仓库管理端（现有）
  ├─ portal/            客户门户（新建）★
  ├─ backend/           共用后端 API
  └─ shared/            共享 UI 组件库（可选，后期提取）

portal/ 技术栈:
  - Vue 3 + TypeScript（与 frontend 一致）
  - Tailwind CSS / 同 frontend UI 框架
  - vue-i18n（中/日/英三语）
  - vue-router（独立路由）
  - pinia（独立 store）

独立部署:
  - portal.zelix.com（客户访问）
  - app.zelix.com（仓库端）
  - api.zelix.com（共用 API）
```

## 4. 与仓库端的关系 / 倉庫管理端との関係

```
客户门户操作                    仓库端看到
─────────────────────────────────────────
客户创建入库预定        →    入库预定列表出现新单
客户上传 FBA 标         →    预定详情可查看/打印
客户确认差异            →    差异状态更新为"已确认"
客户补传 FBA 标         →    状态从"等待FBA标"推进

仓库端操作                    客户门户看到
─────────────────────────────────────────
仓库受付                →    状态变"已受付"
仓库提交差异            →    差异明细出现 + 通知
仓库完成作业            →    作业进度更新 + 费用确定
仓库出货                →    状态变"已出货" + 追踪号
仓库月末结算            →    账单出现 + 通知
```
