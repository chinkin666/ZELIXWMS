# 01 需求文档 / 要件定義書

> 3PL + FBA 通过型业务系统增强 / 3PL + FBA 通過型業務システム強化
> 基于 SOP 文档映射系统需求 / SOP に基づくシステム要件マッピング

## 1. 背景 / 背景

ZELIX WMS 已具备基础的入库、库存、出库、FBA 纳品计划、退货、计费功能。但对照日本 3PL + FBA 通过型业务 SOP（管理版 + 仓库作业版），仍有多项核心流程缺乏系统支撑，依赖线下纸质表单或口头协调。

ZELIX WMS は基本的な入庫、在庫、出庫、FBA 納品プラン、返品、請求機能を備えている。しかし日本 3PL + FBA 通過型業務 SOP と照合すると、複数のコアプロセスにシステム支援が不足しており、紙帳票や口頭調整に依存している。

本文档将 SOP 中的业务要求逐项映射为系统功能需求，并标注优先级和当前覆盖状态。

本書は SOP の業務要件をシステム機能要件に逐一マッピングし、優先度と現状カバー率を示す。

### 2.3 范围外 / スコープ外

以下内容由线下管理制度或外部合作方负责，不纳入系统开发范围：

以下は社内管理制度または外部パートナーが対応し、システム開発スコープ外とする：

- 保险投保（合同条款）/ 保険加入（契約条項）
- IOR/ACP/消费税处理（通关伙伴）/ IOR/ACP/消費税（通関パートナー）
- 赔偿理赔流程（合同 + 线下，后期可做工单）/ 賠償処理（契約 + オフライン、後期で工単化可能）
- 设备采购规格（运营采购）/ 設備調達（運営調達）
- 非标准服务拒绝清单（客户准入审核）/ 非標準サービス拒否リスト（顧客審査）
- SOP 版本管理（Phase 4+ 考虑系统化）/ SOP バージョン管理（Phase 4+ でシステム化検討）
