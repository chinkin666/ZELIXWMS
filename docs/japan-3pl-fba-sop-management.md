# 日本本土 3PL + 中国来日 FBA 通过型业务 SOP（管理版） / 日本国内 3PL + 中国発日本向け FBA 通過型業務 SOP（管理版）

## 1. 文档目的 / 文書目的

本文件用于管理层、业务负责人、项目负责人统一理解日本本土 `3PL` 与“中国发货到日本后转 `Amazon.co.jp FBA`”业务的标准流程、职责边界、报价结构、风险控制与考核指标。

本書は、管理層・営業責任者・プロジェクト責任者が、日本国内 `3PL` と「中国から日本へ輸送後、`Amazon.co.jp FBA` へ転送する業務」の標準フロー、責任分界、料金構成、リスク管理、評価指標を統一理解するために使用する。

## 2. 业务定义 / 業務定義

### 2.1 日本本土 3PL / 日本国内 3PL

- 为客户提供 `入库`、`保管`、`检品`、`流通加工`、`出库`、`配送` 等仓配服务。
- 适用于日本本地电商、批发、门店补货、`D2C`、平台仓前置仓等场景。

- 顧客に対して `入庫`、`保管`、`検品`、`流通加工`、`出庫`、`配送` などの倉配サービスを提供する。
- 日本国内 EC、卸売、店舗補充、`D2C`、プラットフォーム向け前置倉庫などに適用される。

### 2.2 中国来日 FBA 通过型 / 中国発日本向け FBA 通過型

- 业务路径：`中国出货` -> `日本清关` -> `日本中转仓/3PL仓` -> `检品/贴标/分箱/合箱/换箱` -> `Amazon.co.jp FBA`
- 本质不是单纯运输，而是 `头程 + 日本落地处理 + FBA纳品` 的组合业务。

- 業務ルート：`中国出荷` -> `日本通関` -> `日本中継倉 / 3PL 倉庫` -> `検品 / ラベル貼付 / 分箱 / 合箱 / 箱替え` -> `Amazon.co.jp FBA`
- 本質は単純な輸送ではなく、`幹線輸送 + 日本国内処理 + FBA 納品` を組み合わせた業務である。

- 套装组合、简单加工 / セット組み・簡易流通加工
- FBA 派送 / FBA 向け配送
- 退货整备 / 返品再整備

### 3.2 非标准服务 / 非標準サービス

## 5. 标准流程 / 標準フロー

### 5.1 客户准入 / 顧客審査

1. 收集客户基础资料与店铺信息  
   顧客基本情報と店舗情報を収集する
2. 确认业务模式：`本土3PL`、`FBA通过型`、`混合型`  
   業務形態を確認する：`国内3PL`、`FBA通過型`、`混合型`
3. 审核商品是否涉及进口限制、品牌风险、平台限制  
   商品が輸入規制、ブランドリスク、プラットフォーム制限に該当しないか確認する

### 5.2 报价与建档 / 見積・マスタ登録

1. 建立客户档案、`SKU` 主数据、标签规则、箱规信息  
   顧客台帳、`SKU` マスタ、ラベル規則、梱包規格を登録する
2. 报价拆分为：
   見積を以下に分解する：
- 头程费用 / 幹線輸送費
- 清关及代理费 / 通関・代理費
- 入仓费 / 入庫費
- 仓储费 / 保管費
- 加工作业费 / 加工作業費
- FBA 派送费 / FBA 配送費
3. 约定 `SLA` 与异常反馈时效  
   `SLA` と異常報告の時限を合意する

### 5.3 订舱与出运 / 輸送手配・出荷

1. 审核发票、装箱单、运单资料  
   インボイス、梱包明細、輸送書類を確認する
2. 确认进口主体、报关代理、税费承担方  
   輸入者、通関代理、税費負担者を確認する
3. 对客户下发出货规范：
   顧客へ出荷基準を通知する：
- 每箱编号连续 / 箱番号を連番管理
- 箱内 `SKU` 与数量清晰 / 箱内 `SKU` と数量を明確化
- 高风险品单独申报 / 高リスク商品は個別申告

### 5.4 清关与入仓 / 通関・入庫

1. 追踪到港与放行时间  
   到着・通関許可時刻を追跡する
2. 放行后提货入仓  
   許可後に引取し倉庫へ搬入する
3. 仓库核对箱数、重量、体积、外观  
   倉庫で箱数、重量、容積、外観を確認する
4. 对数量、破损、单证异常进行拍照并反馈  
   数量差異、破損、書類異常を撮影し報告する

### 5.5 检品与 FBA 预处理 / 検品・FBA 前処理

1. 按客户要求执行抽检或全检  
   顧客要件に応じて抜取検品または全数検品を行う
2. 按 Amazon 要求执行贴标、套袋、防漏、防刮、组合等处理  
   Amazon 要件に従ってラベル貼付、袋入れ、防漏、防傷、セット組みを行う
3. 复核 `SKU`、`FNSKU`、箱号与数量是否一致  
   `SKU`、`FNSKU`、箱番号、数量の一致を再確認する

### 5.6 建立纳品计划并发往 FBA / 納品プラン作成・FBA 出荷

1. 获取客户 `Seller Central` 纳品计划  
   顧客の `Seller Central` 納品プランを取得する
2. 锁定 FBA 仓、箱数、箱唛、配送方式  
   FBA 倉庫、箱数、箱ラベル、配送方法を確定する
3. 发运并回传追踪号  
   出荷し追跡番号を返却する
4. 跟踪签收、接收、上架差异  
   配達完了、受領、棚入れ差異を追跡する

### 5.7 退货逆向 / 返品逆物流

1. 接收退货  
   返品を受領する
2. 分类为可售、整备后可售、不可售  
   再販可、整備後再販可、再販不可に分類する
3. 对可售商品重新整备并决定去向：`FBA`、`FBM`、退回中国、报废  
   再販可能品を再整備し、`FBA`、`FBM`、中国返送、廃棄のいずれかを決定する

## 6. 操作红线 / 運用上の禁止事項

- 没有 `SKU` 主数据，不进入正式作业  
  `SKU` マスタがない場合、正式作業に入らない
- 没有装箱单，不做正式入库  
  梱包明細がない場合、正式入庫しない
- 没有 FBA 纳品计划，不发 FBA  
  FBA 納品プランがない場合、FBA へ出荷しない
- 涉嫌侵权或受限商品，暂停操作并升级确认  
  権利侵害疑義品または規制品は作業停止し、上位確認を行う
- 所有异常必须 `当日拍照、当日反馈、留档可追溯`  
  すべての異常は `当日撮影・当日報告・追跡可能な記録保管` を必須とする

## 7. 风险控制 / リスク管理

### 7.1 常见风险 / 主要リスク

- `SKU` 与 `FNSKU` 混贴
- 箱唛贴错导致错发 FBA 仓
- 中国端装箱单不准导致入库差异
- 品名申报不规范导致清关风险
- 受限品未提前确认导致扣关或拒收

- `SKU` と `FNSKU` の貼り間違い
- 箱ラベル誤貼付による FBA 倉誤配送
- 中国側梱包明細不一致による入庫差異
- 申告品名不備による通関リスク
- 規制品未確認による差止め・受領拒否

### 7.2 控制措施 / 管理対策

- 建立 `SKU/FNSKU` 双人复核机制  
  `SKU/FNSKU` のダブルチェック体制を設ける
- 出货前核对 `纳品计划-箱号-箱唛-追踪号` 四项一致  
  出荷前に `納品プラン・箱番号・箱ラベル・追跡番号` の一致を確認する
- 所有高风险品类实行准入审批  
  高リスク商材は事前承認制とする
- 客户异常授权写入合同或作业确认书  
  異常処理権限を契約または作業確認書に明記する

## 9. KPI 指标 / KPI 指標

- 入仓准确率 >= `99.5%`

- FBA 错发率 <= `0.2%`
- 异常反馈时效 <= `当日`
- 到仓后出货时效：
  - 标准件 `24-48h`
  - 需加工件 `48-72h`

- 入庫精度 >= `99.5%`
- ラベル精度 >= `99.8%`
- FBA 誤配送率 <= `0.2%`
- 異常報告時限 <= `当日`
- 入庫後出荷リードタイム：
  - 標準品 `24-48h`
  - 加工品 `48-72h`

## 12. 保险与赔偿标准 / 保険・賠償基準

### 12.1 赔偿范围 / 賠償範囲

- 仓库责任导致的货损、丢件、错发
  倉庫側過失による破損、紛失、誤出荷
- 贴标错误导致的 FBA 拒收
  ラベル貼付ミスによる FBA 受領拒否

### 12.2 赔偿上限 / 賠償上限

- 单件赔偿不超过 `申报价值` 或 `合同约定上限`（取较低者）
  1 件あたり `申告価格` または `契約上限額` のいずれか低い方
- 整票赔偿不超过 `该票作业费的 3 倍`（除非另有约定）
  1 案件あたり `当該作業費の 3 倍` を上限とする（別途合意がない限り）
- 不可抗力（自然灾害、运输途中损坏）不在赔偿范围内
  不可抗力（自然災害、輸送中の破損）は賠償対象外

### 12.3 理赔流程 / 賠償処理フロー

1. 客户提交异常申报（附照片与数量证明）
   顧客が異常申告を提出する（写真・数量証拠を添付）
2. 仓库 `48h` 内完成调查并出具调查报告
   倉庫が `48h` 以内に調査完了し報告書を発行する
3. 双方确认责任归属
   双方で責任所在を確認する
4. 确认后 `30日` 内完成赔付或抵扣
   確認後 `30日` 以内に賠償または相殺を完了する

### 12.4 保险建议 / 保険推奨

- 建议客户自行投保货物运输险
  顧客側で貨物輸送保険の加入を推奨する
- 仓库方应投保仓储综合险（火灾、水灾、盗窃）
  倉庫側は総合倉庫保険（火災、水害、盗難）に加入すること
- 高价值商品（单件 >= `5万円`）建议额外投保
  高額商品（1 件 >= `5万円`）は追加保険を推奨する

## 13. Amazon FBA 规格约束 / Amazon FBA 仕様制約

> 以下规格以 Amazon.co.jp 最新要求为准，需定期核查更新。
> 以下仕様は Amazon.co.jp の最新要件に準拠し、定期的に確認更新すること。

### 13.1 箱规限制 / 箱規格制限

- 标准箱重量上限：`15kg`（混合 SKU）/ `30kg`（单一 SKU 且标注"重量物"）
  標準箱重量上限：`15kg`（混合 SKU）/ `30kg`（単一 SKU かつ「重量物」表示あり）
- 箱最长边不超过 `63.5cm`，三边合计不超过 `150cm`
  箱の最長辺 `63.5cm` 以内、三辺合計 `150cm` 以内
- 超规格箱需提前申请 `大型商品` 纳品
  規格超過の場合は `大型商品` 納品として事前申請が必要

### 13.2 FNSKU 贴标要求 / FNSKU ラベル要件

- 每件商品必须有可扫描的 `FNSKU` 条码
  各商品にスキャン可能な `FNSKU` バーコードを貼付する
- 覆盖原有条码（UPC/JAN/EAN）不得外露
  元のバーコード（UPC/JAN/EAN）を覆い隠すこと
- 标签尺寸推荐 `3cm x 5cm`，不得折叠或起皱
  ラベルサイズは `3cm x 5cm` 推奨、折れ・しわ不可

### 13.3 包装要求 / 包装要件

- 液体商品必须密封 + 防漏袋
  液体商品はシール + 防漏袋必須
- 窒息风险商品（薄膜袋装）须贴防窒息警告标签
  窒息リスク品（フィルム袋入り）は窒息防止警告ラベル必須
- 尖锐商品必须有保护包装
  鋭利商品は保護梱包必須
- 套装商品须标注"套装勿拆"（`Set / セット 開封厳禁`）
  セット品は「Set / セット 開封厳禁」の表示必須

### 13.4 版本管理 / バージョン管理

- 每季度核查一次 Amazon FBA 最新 Prep 要求
  四半期ごとに Amazon FBA 最新 Prep 要件を確認する
- 规格变更后 `7日` 内更新本文档并通知全员
  仕様変更後 `7日` 以内に本書を更新し全員に通知する
- 核查信息源：`Seller Central > ヘルプ > FBA商品の梱包要件`
  確認元：`Seller Central > ヘルプ > FBA商品の梱包要件`

## 14. 多仓纳品处理 / 複数 FC 納品対応

### 14.1 背景 / 背景

Amazon 经常将一个纳品计划拆分到多个 FC（Fulfillment Center），客户无法指定单一仓库。
Amazon は 1 つの納品プランを複数 FC に分割することが多く、顧客は単一倉庫を指定できない。

### 14.2 处理流程 / 対応フロー

1. 接收到纳品计划后，确认拆分仓数量与各仓 SKU/箱数分配
   納品プラン受領後、分割先 FC 数と各 FC の SKU・箱数配分を確認する
2. 按各 FC 要求分别分拣、贴箱唛、打包
   各 FC 要件に従って仕分け、箱ラベル貼付、梱包を行う
3. 不同 FC 的货物必须物理隔离，严禁混装
   異なる FC 向けの貨物は物理的に分離し、混載厳禁
4. 分别安排配送，每个 FC 独立追踪
   FC ごとに配送手配し、個別に追跡する

### 14.3 费用规则 / 費用ルール

- 多仓纳品产生的额外分拣与配送费用，按实际仓数收取
  複数 FC 納品による追加仕分け・配送費は、実際の FC 数に応じて請求する
- 建议客户开通 `Inventory Placement Service`（Amazon 合仓服务）以降低成本
  コスト削減のため `Inventory Placement Service`（Amazon 合倉サービス）の利用を推奨する

## 15. IOR（进口主体）与税务处理 / IOR（輸入者）と税務処理

### 15.1 常见模式 / 一般的な形態

| 模式 / 形態 | 进口主体 / 輸入者 | 消费税处理 / 消費税処理 | 适用场景 / 適用場面 |
|---|---|---|---|
| A：客户自有日本法人 | 客户日本法人 / 顧客日本法人 | 法人自行申报抵扣 / 法人が自ら申告控除 | 大型卖家 / 大規模セラー |
| B：客户为非居住者 | 客户（非居住者）/ 顧客（非居住者） | 需指定 ACP（税关事务代理）/ ACP（税関事務代理人）の指定が必要 | 中小卖家 / 中小セラー |
| C：代理进口 | 3PL 或贸易公司 / 3PL または貿易会社 | 代理方承担纳税义务 / 代理側が納税義務を負う | 需要谨慎评估 / 慎重評価が必要 |

### 15.2 ACP（税关事务代理）/ ACP（税関事務代理人）

- 非居住者在日本进口必须指定 ACP 或通关业者
  非居住者が日本で輸入する場合、ACP または通関業者の指定が必須
- ACP 代行税关手续，但纳税义务仍属进口者
  ACP は税関手続を代行するが、納税義務は輸入者に帰属
- 建议客户在开始出货前完成 ACP 签约
  出荷開始前に ACP 契約を完了することを推奨

### 15.3 消费税相关 / 消費税関連

- 进口消费税税率：`10%`（一般商品）/ `8%`（食品等轻减税率对象）
  輸入消費税率：`10%`（一般商品）/ `8%`（食品等軽減税率対象）
- 法人可通过确定申告抵扣进口消费税
  法人は確定申告で輸入消費税を控除可能
- 非居住者无法直接抵扣，需通过 ACP 或日本法人处理
  非居住者は直接控除不可、ACP または日本法人を通じて処理する

### 15.4 我方立场 / 当社の立場

- 我方不提供税务或法律意见
  当社は税務・法務のアドバイスを提供しない
- 进口主体、税务处理由客户自行决定并承担责任
  輸入者・税務処理は顧客が自ら決定し責任を負う
- 我方仅配合客户或其通关代理完成仓库端操作
  当社は顧客またはその通関代理に協力し、倉庫側作業のみを行う

## 16. 大促与旺季应对 / セール・繁忙期対応

### 16.1 主要大促节点 / 主要セール時期

- `Prime Day`（通常 7 月）
- `Black Friday / Cyber Monday`（11 月下旬）
- `年末商戦`（12 月）
- `新生活応援セール`（3 月）

### 16.2 事前准备 / 事前準備

- 大促前 `30日`：收集客户预估入库量，评估仓容与人力
  セール `30日` 前：顧客の入庫予測量を収集し、倉庫容量と人員を評価する
- 大促前 `14日`：确认 FBA 入库截止日（Amazon 会公布），通知客户提前发货
  セール `14日` 前：FBA 入庫締切日（Amazon が公表）を確認し、顧客に早期出荷を通知する
- 大促前 `7日`：冻结非紧急入库，优先处理 FBA 纳品
  セール `7日` 前：非緊急入庫を凍結し、FBA 納品を優先処理する

### 16.3 运营调整 / 運営調整

- 临时增员计划（提前 `14日` 确认）
  臨時増員計画（`14日` 前に確定）
- 延长作业时间或增加班次
  作業時間延長またはシフト増設
- 仓容预警线：使用率达 `80%` 时启动临时仓方案
  倉庫容量警戒線：利用率 `80%` で臨時倉庫を手配

### 16.4 大促后 / セール後

- 退货高峰预案（大促后 `2-4周`）
  返品ピーク対策（セール後 `2-4週`）
- 库存滞留预警与客户沟通
  滞留在庫の警告と顧客への連絡
