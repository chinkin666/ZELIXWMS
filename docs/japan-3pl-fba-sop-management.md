# 日本本土 3PL + 中国来日 FBA 通过型业务 SOP（管理版） / 日本国内 3PL + 中国発日本向け FBA 通過型業務 SOP（管理版）

## 1. 文档目的 / 文書目的

本文件用于管理层、业务负责人、项目负责人统一理解日本本土 `3PL` 与“中国发货到日本后转 `Amazon.co.jp FBA`”业务的标准流程、职责边界、报价结构、风险控制与考核指标。

本書は、管理層・営業責任者・プロジェクト責任者が、日本国内 `3PL` と「中国から日本へ輸送後、`Amazon.co.jp FBA` へ転送する業務」の標準フロー、責任分界、料金構成、リスク管理、評価指標を統一理解するために使用する。

## 2. 业务定义 / 業務定義

### 2.1 日本本土 3PL / 日本国内 3PL

- 为客户提供 `入库`、`保管`、`检品`、`流通加工`、`出库`、`配送` 等仓配服务。
- 适用于日本本地电商、批发、门店补货、`D2C`、平台仓前置仓等场景。

- 顧客に対して `入庫`、`保管`、`検品`、`流通加工`、`出庫`、`配送` などの倉配サービスを提供する。
- 日本国内 EC、卸売、店舗補充、`D2C`、プラットフォーム向け前置倉庫などに適用される。

### 2.2 中国来日 FBA 通过型 / 中国発日本向け FBA 通過型

- 业务路径：`中国出货` -> `日本清关` -> `日本中转仓/3PL仓` -> `检品/贴标/分箱/合箱/换箱` -> `Amazon.co.jp FBA`
- 本质不是单纯运输，而是 `头程 + 日本落地处理 + FBA纳品` 的组合业务。

- 業務ルート：`中国出荷` -> `日本通関` -> `日本中継倉 / 3PL 倉庫` -> `検品 / ラベル貼付 / 分箱 / 合箱 / 箱替え` -> `Amazon.co.jp FBA`
- 本質は単純な輸送ではなく、`幹線輸送 + 日本国内処理 + FBA 納品` を組み合わせた業務である。

### 3.1 标准服务 / 標準サービス

- 头程订舱对接 / 幹線輸送手配
- 清关资料审核 / 通関資料チェック
- 日本仓入库收货 / 日本倉庫での入庫受領
- 检品与拍照 / 検品・撮影
- 商品贴标、箱唛贴附 / 商品ラベル・箱ラベル貼付
- 分箱、合箱、换箱 / 分箱・合箱・箱替え
- 套装组合、简单加工 / セット組み・簡易流通加工
- FBA 派送 / FBA 向け配送
- 退货整备 / 返品再整備

### 3.2 非标准服务 / 非標準サービス

- 食品、化妆品、医疗器械、电池、带磁、危险品
- 品牌授权审查、知识产权敏感商品
- 需预约检验或额外许可的进口商品

- 食品、化粧品、医療機器、電池、磁性品、危険物
- ブランド許諾審査、知的財産に敏感な商品
- 検査予約や追加許認可が必要な輸入品

## 5. 标准流程 / 標準フロー

### 5.1 客户准入 / 顧客審査

1. 收集客户基础资料与店铺信息  
   顧客基本情報と店舗情報を収集する
2. 确认业务模式：`本土3PL`、`FBA通过型`、`混合型`  
   業務形態を確認する：`国内3PL`、`FBA通過型`、`混合型`
3. 审核商品是否涉及进口限制、品牌风险、平台限制  
   商品が輸入規制、ブランドリスク、プラットフォーム制限に該当しないか確認する

### 5.2 报价与建档 / 見積・マスタ登録

1. 建立客户档案、`SKU` 主数据、标签规则、箱规信息  
   顧客台帳、`SKU` マスタ、ラベル規則、梱包規格を登録する
2. 报价拆分为：
   見積を以下に分解する：
- 头程费用 / 幹線輸送費
- 清关及代理费 / 通関・代理費
- 入仓费 / 入庫費
- 仓储费 / 保管費
- 加工作业费 / 加工作業費
- FBA 派送费 / FBA 配送費
3. 约定 `SLA` 与异常反馈时效  
   `SLA` と異常報告の時限を合意する

### 5.3 订舱与出运 / 輸送手配・出荷

1. 审核发票、装箱单、运单资料  
   インボイス、梱包明細、輸送書類を確認する
2. 确认进口主体、报关代理、税费承担方  
   輸入者、通関代理、税費負担者を確認する
3. 对客户下发出货规范：
   顧客へ出荷基準を通知する：
- 每箱编号连续 / 箱番号を連番管理
- 箱内 `SKU` 与数量清晰 / 箱内 `SKU` と数量を明確化
- 高风险品单独申报 / 高リスク商品は個別申告

### 5.4 清关与入仓 / 通関・入庫

1. 追踪到港与放行时间  
   到着・通関許可時刻を追跡する
2. 放行后提货入仓  
   許可後に引取し倉庫へ搬入する
3. 仓库核对箱数、重量、体积、外观  
   倉庫で箱数、重量、容積、外観を確認する
4. 对数量、破损、单证异常进行拍照并反馈  
   数量差異、破損、書類異常を撮影し報告する

### 5.5 检品与 FBA 预处理 / 検品・FBA 前処理

1. 按客户要求执行抽检或全检  
   顧客要件に応じて抜取検品または全数検品を行う
2. 按 Amazon 要求执行贴标、套袋、防漏、防刮、组合等处理  
   Amazon 要件に従ってラベル貼付、袋入れ、防漏、防傷、セット組みを行う
3. 复核 `SKU`、`FNSKU`、箱号与数量是否一致  
   `SKU`、`FNSKU`、箱番号、数量の一致を再確認する

### 5.6 建立纳品计划并发往 FBA / 納品プラン作成・FBA 出荷

1. 获取客户 `Seller Central` 纳品计划  
   顧客の `Seller Central` 納品プランを取得する
2. 锁定 FBA 仓、箱数、箱唛、配送方式  
   FBA 倉庫、箱数、箱ラベル、配送方法を確定する
3. 发运并回传追踪号  
   出荷し追跡番号を返却する
4. 跟踪签收、接收、上架差异  
   配達完了、受領、棚入れ差異を追跡する

### 5.7 退货逆向 / 返品逆物流

1. 接收退货  
   返品を受領する
2. 分类为可售、整备后可售、不可售  
   再販可、整備後再販可、再販不可に分類する
3. 对可售商品重新整备并决定去向：`FBA`、`FBM`、退回中国、报废  
   再販可能品を再整備し、`FBA`、`FBM`、中国返送、廃棄のいずれかを決定する

## 6. 操作红线 / 運用上の禁止事項

- 没有 `SKU` 主数据，不进入正式作业  
  `SKU` マスタがない場合、正式作業に入らない
- 没有装箱单，不做正式入库  
  梱包明細がない場合、正式入庫しない
- 没有 FBA 纳品计划，不发 FBA  
  FBA 納品プランがない場合、FBA へ出荷しない
- 涉嫌侵权或受限商品，暂停操作并升级确认  
  権利侵害疑義品または規制品は作業停止し、上位確認を行う
- 所有异常必须 `当日拍照、当日反馈、留档可追溯`  
  すべての異常は `当日撮影・当日報告・追跡可能な記録保管` を必須とする

## 7. 风险控制 / リスク管理

### 7.1 常见风险 / 主要リスク

- `SKU` 与 `FNSKU` 混贴
- 箱唛贴错导致错发 FBA 仓
- 中国端装箱单不准导致入库差异
- 品名申报不规范导致清关风险
- 受限品未提前确认导致扣关或拒收

- `SKU` と `FNSKU` の貼り間違い
- 箱ラベル誤貼付による FBA 倉誤配送
- 中国側梱包明細不一致による入庫差異
- 申告品名不備による通関リスク
- 規制品未確認による差止め・受領拒否

### 7.2 控制措施 / 管理対策

- 建立 `SKU/FNSKU` 双人复核机制  
  `SKU/FNSKU` のダブルチェック体制を設ける
- 出货前核对 `纳品计划-箱号-箱唛-追踪号` 四项一致  
  出荷前に `納品プラン・箱番号・箱ラベル・追跡番号` の一致を確認する
- 所有高风险品类实行准入审批  
  高リスク商材は事前承認制とする
- 客户异常授权写入合同或作业确认书  
  異常処理権限を契約または作業確認書に明記する

## 9. KPI 指标 / KPI 指標

- 入仓准确率 >= `99.5%`
- 贴标准确率 >= `99.8%`
- FBA 错发率 <= `0.2%`
- 异常反馈时效 <= `当日`
- 到仓后出货时效：
  - 标准件 `24-48h`
  - 需加工件 `48-72h`

- 入庫精度 >= `99.5%`
- ラベル精度 >= `99.8%`
- FBA 誤配送率 <= `0.2%`
- 異常報告時限 <= `当日`
- 入庫後出荷リードタイム：
  - 標準品 `24-48h`
  - 加工品 `48-72h`

- 本文件是业务运营 `SOP`，不是法律、税务或报关意见书。

- 輸入申告主体、通関代理、税費負担方式は出荷前に確定しなければならない。
- 非居住者、消費税、輸入許可などは、顧客が通関・税務専門家と確認したうえで実行する必要がある。
- 本書は業務運営 `SOP` であり、法律・税務・通関の意見書ではない。
