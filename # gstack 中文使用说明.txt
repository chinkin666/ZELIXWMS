## 一、gstack 是什么？

gstack 是一个 **AI 软件工厂**，它把 Claude Code 变成一个你可以管理的"虚拟工程团队"。

它包含 **15 个专业角色 + 6 个增强工具**，全部以斜杠命令（`/命令`）方式调用。每个命令模拟一个工程团队成员的职责——CEO 评审策略、工程经理锁定架构、设计师审查 UI、QA 用真实浏览器测试、发布工程师负责上线。

---

## 二、核心理念：Sprint 开发流程

gstack 将开发分为 **7 个阶段**，形成一个完整的 Sprint 循环：

```
想 (Think) → 规划 (Plan) → 构建 (Build) → 审查 (Review) → 测试 (Test) → 发布 (Ship) → 复盘 (Reflect)
```

支持 **10-15 个 Sprint 并行推进**，大幅提升开发效率。

## 三、全部命令速查表

### A. 规划阶段（4 个命令）

| 命令 | 角色 | 功能说明 |
|------|------|----------|
| `/office-hours` | 产品顾问 | YC 风格的头脑风暴。描述你的想法，它会重新定义问题、挑战你的假设、提出替代方案。**用于项目启动时。** |
| `/plan-ceo-review` | CEO | 从战略角度审查你的计划。评估 10 个维度：市场适配、技术可行性、范围控制等。**用于确认方向是否正确。** |
| `/plan-eng-review` | 工程经理 | 锁定技术架构。生成数据流图、识别技术债务、确认依赖关系。**用于开始编码前。** |
| `/plan-design-review` | 设计师 | 设计维度评审。检查用户体验、可访问性、响应式设计等方面。**用于确认设计方案。** |

### B. 设计与开发阶段（3 个命令）

| 命令 | 角色 | 功能说明 |
|------|------|----------|
| `/design-consultation` | 设计系统师 | 从零开始生成完整的设计系统（颜色、字体、组件、间距等）。**用于需要统一设计语言时。** |
| `/investigate` | 调试专家 | 系统性根因调试。用假设检验的方式定位 Bug，不是盲目试错。**用于遇到难以排查的问题时。** |
| `/browse` | 浏览器 | 控制真实的 Chromium 浏览器。可以导航、点击、截图、填表单。每条命令约 100ms。**用于需要操作网页时。** |

### C. 审查阶段（2 个命令）

| 命令 | 角色 | 功能说明 |
|------|------|----------|
| `/review` | 高级审查员 | Staff 级别的代码审查。检查代码质量、安全性、性能问题，并自动修复。**用于合并代码前。** |
| `/design-review` | UI/UX 审查员 | 用真实浏览器审查已实现的界面。检查视觉一致性、响应式布局、可访问性。**用于 UI 实现后。** |

### D. 测试阶段（2 个命令）

| 命令 | 角色 | 功能说明 |
|------|------|----------|
| `/qa` | QA 主管 | 用真实浏览器测试 → 发现 Bug → 自动修复 → 再验证。生成原子提交。**用于功能完成后。** |
| `/qa-only` | QA 报告员 | 只测试和报告 Bug，不做任何修改。**用于只想了解问题但不想自动修复时。** |

### E. 发布与维护阶段（3 个命令）

| 命令 | 角色 | 功能说明 |
|------|------|----------|
| `/ship` | 发布工程师 | 自动化发布流程：运行测试、创建 PR、确保 CI 通过。如果没有测试框架会自动搭建。**用于准备上线时。** |
| `/document-release` | 文档工程师 | 发布后自动同步文档：README、API 文档、CHANGELOG 等。**用于发布完成后。** |
| `/retro` | 回顾主持人 | 开发回顾会议。分析过去一周的提交统计、代码质量趋势、改进建议。**用于每周复盘。** |

### F. 增强工具（6 个命令）

| 命令 | 功能说明 |
|------|----------|
| `/codex` | 调用 OpenAI 的模型做独立代码审查（需要 OpenAI API Key）。提供对抗性视角的第二意见。 |
| `/careful` | 开启安全警告模式。在执行危险命令（如 `rm -rf`、`DROP TABLE`、`force-push`）前发出警告。 |
| `/freeze` | 限制编辑范围。只允许修改指定目录内的文件，防止误改其他代码。 |
| `/guard` | 最大安全模式。同时启用 `/careful` + `/freeze`。 |
| `/unfreeze` | 解除 `/freeze` 的编辑限制。 |
| `/gstack-upgrade` | 升级 gstack 到最新版本。 |

## 四、快速上手指南

### 第 1 步：验证安装

gstack 已安装在 `~/.claude/skills/gstack/`，所有命令已注册。直接在 Claude Code 中使用斜杠命令即可。

### 第 2 步：5 分钟体验流程

```
1. 输入 /office-hours     → 描述你要做的东西，让它帮你理清思路
2. 输入 /plan-ceo-review  → 让它从战略角度审查你的计划
3. 输入 /review           → 对你现有代码做一次深度审查
4. 输入 /qa               → 用真实浏览器测试你的网站/应用
```

### 第 3 步：日常开发工作流

```
有新想法    → /office-hours → /plan-ceo-review → /plan-eng-review
开始编码    → 正常写代码
写完代码    → /review（代码审查）
功能完成    → /qa（浏览器测试）
准备上线    → /ship（自动发布）
上线后      → /document-release（更新文档）→ /retro（每周复盘）
遇到 Bug   → /investigate（系统性调试）
```

## 五、浏览器功能详解（/browse）

gstack 内置了一个基于 Playwright 的无头浏览器，这是它最强大的功能之一。

### 基本操作

| 操作 | 命令示例 |
|------|----------|
| 打开网页 | `$B navigate https://example.com` |
| 截图 | `$B screenshot` |
| 点击元素 | `$B click @e3`（@e3 是页面元素的引用编号） |
| 输入文字 | `$B fill @e5 "你好世界"` |
| 获取页面快照 | `$B snapshot`（返回页面的可访问性树） |
| 执行 JS | `$B js document.title` |

### 性能

- 首次启动约 3 秒（启动浏览器进程）
- 后续每条命令约 100-200ms
- 30 分钟无操作自动关闭
- 崩溃后自动恢复

### 常用场景

- QA 测试：自动点击、填表单、验证页面状态
- 截图对比：修改前后的视觉差异
- 响应式测试：不同屏幕尺寸下的显示效果
- 表单测试：自动填写和提交
- 对话框处理：自动处理 alert/confirm/prompt

## 六、安全功能

### /careful 模式

开启后，以下操作会触发警告：
- `rm -rf` 或批量删除
- `DROP TABLE`、`DELETE FROM` 等数据库操作
- `git push --force`、`git reset --hard`
- 修改 CI/CD 配置
- 操作生产环境

### /freeze 模式

限制 Claude 只能编辑指定目录：
```
/freeze src/components    → 只能修改 src/components 目录下的文件
/unfreeze                 → 解除限制
```

### /guard 模式

同时启用 careful + freeze，提供最高安全保障。适用于操作生产环境或关键代码时。

## 七、添加到项目（团队共享）

如果你想让团队成员也能使用 gstack：

```bash
# 在项目根目录执行
cp -Rf ~/.claude/skills/gstack .claude/skills/gstack
rm -rf .claude/skills/gstack/.git
cd .claude/skills/gstack && ./setup
```

这会把 gstack 复制到项目的 `.claude/skills/` 目录中，提交后团队成员 `git clone` 即可使用。

## 八、常见问题

### Q: 需要什么前提条件？
- Claude Code（必须）
- Git（必须）
- Bun v1.0+（必须，Windows 上也可用 Node.js）

### Q: 浏览器功能不工作？
```bash
cd ~/.claude/skills/gstack && ./setup
```
重新运行 setup 即可。

### Q: 如何升级？
在 Claude Code 中输入 `/gstack-upgrade`。

### Q: 收费吗？
完全免费，MIT 开源许可证，永远免费。

### Q: /codex 需要什么？
需要设置 `OPENAI_API_KEY` 环境变量。这是唯一需要额外 API Key 的命令。

### Q: 支持哪些 AI 工具？
Claude Code、Codex、Gemini CLI、Cursor 都支持。

## 十、技术架构简述

- **浏览器引擎**：基于 Playwright 的 Chromium 无头浏览器
- **通信方式**：CLI 客户端通过 localhost HTTP 与持久化 Bun 服务器通信
- **安全模型**：仅限 localhost 访问，UUID 令牌认证
- **性能**：100ms 级别的命令响应（首次启动 ~3 秒）
- **SKILL.md 标准**：所有技能以 Markdown 文件定义，兼容多种 AI 代理
